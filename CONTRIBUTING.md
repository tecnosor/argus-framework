# Contributing to Argus Framework

Thank you for your interest in contributing to Argus Framework!

## How to Contribute

### Reporting Issues

- Use the [Bug Report](https://github.com/tecnosor/argus-framework/issues/new?template=bug_report.md) template for bugs.
- Use the [Feature Request](https://github.com/tecnosor/argus-framework/issues/new?template=feature_request.md) template for enhancements.
- Use the [Security Vulnerability](https://github.com/tecnosor/argus-framework/issues/new?template=security_vulnerability.md) template for security issues.

### Pull Requests

1. **Fork the repository** and create your branch from `develop`.
2. **Follow the existing code style** and formatting conventions.
3. **Write clear, concise commit messages** following [Conventional Commits](https://www.conventionalcommits.org/).
4. **Ensure your PR description** clearly describes the problem and solution. Include relevant issue numbers.
5. **All PRs must be approved by the repository owner** (`@tecnosor`) before merging.
6. **Ensure CI checks pass** before requesting review.

### Review Process

- The repository owner (`@tecnosor`) is the sole approver for all changes.
- Reviews are typically completed within 7 days.
- Please be responsive to feedback and requested changes.

### Code of Conduct

All contributors are expected to adhere to our [Code of Conduct](CODE_OF_CONDUCT.md).

## Questions?

If you have questions, open a [GitHub Discussion](https://github.com/tecnosor/argus-framework/discussions) or contact the maintainer directly.
